## Features

- Add, view, update, and delete tasks
- Persistent storage using SQLite
- Filter tasks by:
  - Status (pending / completed)
  - Priority (low / medium / high)
- Modular architecture (models, services, db, utils, ui)
- Strong input validation
- Database-level integrity constraints
- Safe handling of edge cases (Windows DB locking, invalid input)
- Developer-friendly reset utility

## Task Model

Each task includes:
- `task_id` (auto-generated by SQLite)
- `title`
- `description`
- `status` (`pending` / `completed`)
- `priority` (`low` / `medium` / `high`)
- `created_at` (timestamp)

## Resetting the Database (Development Only)

When the **Developer Mode** is enabled, user is prompted to reset the database at the start of the application.

To enable Developer Mode:
- Set `DEV_MODE = True` in the main.py at constants section
